# ⚠️ خطأ: جدول الرسائل غير موجود

## المشكلة
الخطأ الذي ظهر يعني أن جدول `messages` غير موجود في قاعدة البيانات.

## الحل: تطبيق SQL في Supabase

### الخطوة 1: افتح Supabase
1. اذهب إلى https://supabase.com
2. افتح مشروعك
3. من القائمة الجانبية، اختر **SQL Editor**

### الخطوة 2: انسخ الكود التالي
انسخ هذا الكود بالكامل:

```sql
-- إنشاء جدول الرسائل
CREATE TABLE IF NOT EXISTS messages (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  sender_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  receiver_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  subject TEXT NOT NULL,
  body TEXT NOT NULL,
  is_read BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  read_at TIMESTAMPTZ,
  
  CONSTRAINT messages_sender_receiver_check CHECK (sender_id != receiver_id)
);

-- إنشاء الفهارس
CREATE INDEX IF NOT EXISTS idx_messages_sender ON messages(sender_id);
CREATE INDEX IF NOT EXISTS idx_messages_receiver ON messages(receiver_id);
CREATE INDEX IF NOT EXISTS idx_messages_created_at ON messages(created_at DESC);
CREATE INDEX IF NOT EXISTS idx_messages_is_read ON messages(is_read);

-- تفعيل RLS
ALTER TABLE messages ENABLE ROW LEVEL SECURITY;

-- حذف السياسات القديمة إن وجدت
DROP POLICY IF EXISTS "Users can view their own messages" ON messages;
DROP POLICY IF EXISTS "Users can send messages" ON messages;
DROP POLICY IF EXISTS "Receivers can mark messages as read" ON messages;
DROP POLICY IF EXISTS "Users can delete their messages" ON messages;

-- سياسة القراءة
CREATE POLICY "Users can view their own messages"
  ON messages FOR SELECT
  USING (
    auth.uid() = sender_id OR 
    auth.uid() = receiver_id
  );

-- سياسة الإنشاء
CREATE POLICY "Users can send messages"
  ON messages FOR INSERT
  WITH CHECK (auth.uid() = sender_id);

-- سياسة التحديث
CREATE POLICY "Receivers can mark messages as read"
  ON messages FOR UPDATE
  USING (auth.uid() = receiver_id)
  WITH CHECK (auth.uid() = receiver_id);

-- سياسة الحذف
CREATE POLICY "Users can delete their messages"
  ON messages FOR DELETE
  USING (
    auth.uid() = sender_id OR 
    auth.uid() = receiver_id
  );

-- حذف الـ trigger القديم إن وجد
DROP TRIGGER IF EXISTS trigger_notify_new_message ON messages;
DROP FUNCTION IF EXISTS notify_new_message();

-- إنشاء trigger للإشعارات
CREATE OR REPLACE FUNCTION notify_new_message()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO notifications (user_id, type, title, message, related_id)
  VALUES (
    NEW.receiver_id,
    'message',
    'رسالة جديدة',
    'لديك رسالة جديدة من ' || COALESCE((SELECT full_name FROM profiles WHERE id = NEW.sender_id), 'مستخدم'),
    NEW.id::text
  );
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_notify_new_message
  AFTER INSERT ON messages
  FOR EACH ROW
  EXECUTE FUNCTION notify_new_message();

-- رسالة نجاح
SELECT 'تم إنشاء جدول الرسائل بنجاح! ✅' as message;
```

### الخطوة 3: شغل الكود
1. الصق الكود في SQL Editor
2. اضغط **Run** أو **Ctrl+Enter**
3. انتظر حتى تظهر رسالة "تم إنشاء جدول الرسائل بنجاح! ✅"

### الخطوة 4: تحقق من النجاح
شغل هذا الأمر للتحقق:

```sql
SELECT 
  table_name,
  column_name,
  data_type
FROM information_schema.columns
WHERE table_name = 'messages'
ORDER BY ordinal_position;
```

يجب أن ترى قائمة بأعمدة الجدول.

### الخطوة 5: جرب إرسال رسالة مرة أخرى
ارجع للتطبيق وحاول إرسال رسالة - يجب أن تعمل الآن! ✅

## إذا استمرت المشكلة

### تحقق من الأخطاء:
1. **خطأ "relation does not exist"** = الجدول غير موجود، أعد تطبيق SQL
2. **خطأ "permission denied"** = مشكلة في RLS، تأكد من تطبيق السياسات
3. **خطأ "foreign key violation"** = جدول profiles غير موجود

### حل سريع للاختبار (بدون RLS):
إذا أردت اختبار النظام بسرعة بدون RLS:

```sql
-- تعطيل RLS مؤقتاً (للاختبار فقط!)
ALTER TABLE messages DISABLE ROW LEVEL SECURITY;
```

⚠️ **تحذير:** لا تستخدم هذا في الإنتاج! أعد تفعيل RLS بعد الاختبار:

```sql
ALTER TABLE messages ENABLE ROW LEVEL SECURITY;
```
